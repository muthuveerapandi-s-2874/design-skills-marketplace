# Advanced Landing Page System Skill

## Purpose

This skill generates high-quality, modern landing pages by combining flexible section patterns, multiple layout variations, and strong design theory principles. The goal is to produce layouts that feel intentional, balanced, and product-ready rather than template-driven. Each output should reflect real-world SaaS or product design quality, where clarity, structure, and visual hierarchy guide the user experience.

---

## When to Use

Use this skill whenever a landing page or UI section is required, especially when the request involves modern layouts, premium design, or multiple layout explorations. It is ideal for generating non-template, structured pages that resemble real product marketing sites.

---

## Core Approach

The system follows a section-first methodology, where each part of the page is treated as a modular unit with a clear purpose. Instead of repeating a single layout style, the system introduces variation across sections while maintaining consistency through spacing, hierarchy, and visual balance. The output should always feel cohesive, with each section contributing to a clear narrative and user journey.

---

## Section System

The page is constructed using a flexible section library that adapts based on context. Hero sections can vary from classic conversion-focused layouts with headline, subtext, and CTAs, to centered minimal layouts, split layouts with user options, visual-first layouts with strong imagery, or card-based heroes that sit inside containers. Supporting sections include announcement bars for quick updates, feature sections presented as grids, tabs, accordions, bento-style layouts, or single highlighted features, along with testimonials, numeric stats, product showcases, and CTA sections. Each section should feel distinct yet connected, avoiding repetition in structure or density.

---

## Layout System

Layouts are not fixed and should adapt across the page. Standard full-width layouts can be used for clarity, while boxed or card-based layouts—similar to the visual approach often seen in Apple Inc.—introduce a more premium and structured feel through containers, rounded corners, and subtle separation. Alternating layouts help create flow by switching between text and visual alignment, and mixed layouts combine multiple styles to avoid monotony. The overall composition should feel dynamic but controlled.

---

## Design Theory & Balance

All layouts must follow strong design principles to ensure visual quality. The 70/30 rule should guide focus, where primary elements like headlines or key visuals dominate attention while supporting elements remain secondary. A 60/40 split between content and visuals can be applied to maintain balance in sections. Visual hierarchy must always be clear, allowing users to quickly scan headlines, understand supporting text, and identify actions. Layouts should follow natural reading patterns such as Z-pattern or F-pattern flows, ensuring intuitive navigation.

Spacing plays a critical role in maintaining clarity, using a consistent rhythm that moves from tight to medium to wide spacing to create breathing room. Section density should alternate between content-heavy and minimal sections to avoid visual fatigue. Contrast should be used intentionally, with high contrast reserved for key actions or messages and lower contrast for supporting elements. Boxed layouts should group content into clean containers with minimal clutter, relying on spacing and structure rather than excessive decoration.

Repetition should be used to maintain consistency in components, while variation in layout keeps the design engaging. Most importantly, each section must communicate a single clear idea, avoiding the common mistake of mixing multiple messages within the same space.

---

## Layout Intelligence

The system should actively avoid repetitive patterns and instead mix different section styles across the page. Every section must have a focal point, guiding the user’s attention naturally. Spacing, alignment, and hierarchy should remain consistent throughout, ensuring the entire page feels like one cohesive product rather than a collection of disconnected blocks.

---

## AI Behavior Rules

The system should think like a product designer rather than a template generator. It must avoid generic layouts, ensure each section has a clear purpose, and maintain strong visual balance throughout the page. Design theory principles such as the 70/30 rule, hierarchy, and spacing discipline should always be applied. The final output should prioritize clarity, usability, and modern design standards over unnecessary complexity or decoration.

---

## Output Format

Each section should be presented clearly with its role and structure, including a headline, supporting text, key elements, layout direction, and a brief suggestion of the visual approach. The result should feel like a real, production-ready landing page concept that can be directly translated into design or development.

---

## Output Expectations

The final output should feel like a modern SaaS product page with strong visual balance, thoughtful section variety, and clear communication. It should be structured, scannable, and ready for implementation in design tools or a builder system, without appearing repetitive or template-based.
